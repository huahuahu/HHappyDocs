# 资料库入口卡片化设计

## 目标

将 iOS“资料库”首页从三行系统列表改为三张卡片式入口，让“标签”“参与者”“图表”更像平级的资料库功能，同时保持当前导航目标不变。

布局必须根据当前容器的可用空间和 Dynamic Type 自动在三列与单列之间切换，不依赖设备型号、屏幕固定宽度或 `horizontalSizeClass`，并且不出现两列布局、`2 + 1` 孤立卡片或某个入口比其他入口更宽的情况。

## 当前行为

`LibraryView` 在 `NavigationStack` 内使用 `List` 遍历 `LibraryEntry.allCases`，每一行通过 `NavigationLink` 进入已有的标签、参与者或图表页面。

当前首页只展示入口名称和图标，不展示标签与参与者数量。三个入口的目标页面、深链处理和资料库导航栈均已存在，本次不修改这些行为。

## 已确认的产品决策

- 资料库首页只保留一种卡片式展示，不增加列表与网格切换开关。
- 三个入口业务权重相同，正常状态下卡片同宽、同高并排展示。
- 空间不足时，三个入口整体切换成单列卡片；不存在两列中间状态。
- 普通 Dynamic Type 且空间足够时，外部使用三列，每张卡片内部使用 vertical 内容方向。
- 普通 Dynamic Type 且空间不足时，外部使用完整单列，每张卡片内部使用 horizontal 内容方向。
- `dynamicTypeSize.isAccessibilitySize == true` 时，外部强制使用完整单列，每张卡片内部使用 vertical 内容方向。
- 标签和参与者卡片展示实时数量，图表卡片展示静态说明“查看记录趋势”。
- 整张卡片均可点击，并继续使用现有 `HDiaryDestination.libraryEntry` 导航。

## 页面结构与视觉层级

`List` 替换为可滚动的卡片容器，页面继续使用当前的大标题“资料库”。卡片区域设置统一的水平边距和卡片间距，并在宽屏上将最大内容宽度限制为 `720pt` 后居中，避免 iPad 或横屏中卡片被无限拉宽。

每张卡片包含：

1. 现有 `LibraryEntry.symbol` 对应的图标。
2. 现有本地化入口名称。
3. 简短的本地化摘要。
4. 表示可以继续进入的 chevron。

图标继续使用 App 的 tint 色，卡片背景、文字颜色和按压反馈使用系统语义样式，以自动适配浅色、深色和高对比度环境。普通字号三列与 Accessibility 单列的卡片内部采用 vertical 内容方向；只有普通字号因空间不足回退的单列卡片内部采用 horizontal 内容方向。两种内容方向的圆角、背景和信息内容保持一致。

## 响应式布局

使用 `ViewThatFits(in: .horizontal)` 在以下两个候选布局中按顺序选择：

1. 首选布局：一个包含单个 `GridRow` 的 `Grid`，三张等宽卡片同排。
2. 回退布局：一个包含三张整行卡片的 `VStack`。

首选布局中的每张卡片设置由 `@ScaledMetric(relativeTo: .body)` 提供的最小可读宽度，初始基准值为 `104pt`。卡片间距初始为 `12pt`，页面水平边距初始为 `16pt`。`ViewThatFits` 根据三列候选布局的理想宽度是否能容纳在当前水平提议中决定是否回退，不读取 `UIScreen.main.bounds`，也不判断 iPhone 或 iPad 型号。

最小卡片宽度会随 Dynamic Type 放大，使文字变大时三列布局更早回退。普通 Dynamic Type 下，三列候选使用 vertical 卡片；若三列候选放不下，完整单列回退候选使用 horizontal 卡片。

若 `dynamicTypeSize.isAccessibilitySize` 为真，则直接使用完整单列布局，不再尝试三列，并将每张卡片内部改为 vertical 内容方向。这可以让标题和摘要使用整张卡片的可用宽度，避免 horizontal 排列把英文 `Tag`、`20 tags` 等内容挤成单字或两字宽的极窄文字列。

外部布局在任何 Dynamic Type 下都只允许一列或三列；禁止两列、`2 + 1`、孤立空位或第三张卡片更宽。卡片内容方向的选择不改变 `720pt` 最大内容宽度、`@ScaledMetric` 缩放度量或 `ViewThatFits` 的普通字号测量规则。

`ViewThatFits` 的两个候选布局会各自构建卡片子树。当前卡片仅包含无本地状态的 `NavigationLink`，切换布局不会丢失用户输入或业务状态。如果以后卡片内部新增本地交互状态，再评估改用保持子视图身份的 `AnyLayout` 或自定义 `Layout`。

## 组件与数据流

### `LibraryView`

`LibraryView` 继续负责导航栈、深链处理和三个入口的组合，同时通过 SwiftData `@Query` 观察 `Tag` 与 `Participant` 集合。数组数量变化后，卡片摘要随 SwiftData 更新自动刷新。

本次只读取标签和参与者数量。图表卡片不查询或绘制假趋势数据，避免用装饰曲线暗示不存在的统计结果。

### `LibraryEntryCard`

新增独立的 `LibraryEntryCard` 视图，只负责单张卡片的呈现。它接收 `LibraryEntry` 和对应的 `LocalizedStringResource` 摘要，不直接访问 SwiftData，也不决定导航目标。

卡片本身作为 `NavigationLink` 的标签使用。图标属于辅助视觉，VoiceOver 的主要信息由入口标题和摘要组成，避免重复朗读符号名称。

现有 `LibraryEntryCell` 在新页面不再使用，实施时删除该类型，避免同时保留两套入口外观。

### `LibraryEntryLayoutPolicy`

`LibraryEntryLayoutPolicy` 同时封装 Accessibility 强制单列判断，以及单列卡片的内容方向：普通 Dynamic Type 返回 `.horizontal`，Accessibility Dynamic Type 返回 `.vertical`。`LibraryEntryDashboard` 只消费该值并复用 `LibraryEntryCard` 已有的两个内容方向分支，不复制卡片视觉实现。

### 本地化摘要

在 `DiaryStringKey` 中增加紧凑的资料库入口摘要：

- 标签：本地化的“`n` 个标签”。
- 参与者：本地化的“`n` 位参与者”。
- 图表：本地化的“查看记录趋势”。

数量为零时仍显示本地化的零值摘要，不隐藏摘要区域，因此卡片结构不会因数据为空而跳动。英文和其他语言允许摘要自然换行，但三列卡片的入口标题不得截断。

## 无障碍与交互

- 使用语义字体并支持完整 Dynamic Type。
- 卡片最小尺寸和内部边距随必要的度量缩放，点击区域不小于系统建议尺寸。
- 整张卡片使用 `NavigationLink`，不通过 `onTapGesture` 模拟按钮。
- VoiceOver 按“标签、数量”“参与者、数量”“图表、查看记录趋势”的顺序朗读。
- 不仅依靠橙色区分入口；每张卡片同时具有不同图标和文字标题。
- Accessibility 单列卡片内部使用 vertical 内容方向；标题与摘要都必须完整显示，不得截断，也不得形成单字或两字宽的极窄文字列。
- 旋转设备、进入或退出 iPad 分屏以及改变 Dynamic Type 时，布局可以在三列和单列间重新选择，导航数据不受影响。

## 错误处理与边界情况

标签或参与者为空不是错误，卡片显示零值摘要且仍可进入对应的空状态页面。

资料库首页不执行网络请求，也没有需要单独展示的加载或失败状态。SwiftData 模型容器继续由现有 App 环境提供；若容器本身无法建立，沿用 App 当前的全局数据层错误处理，本页面不新增独立恢复流程。

三列候选布局只有在三张卡片都达到最小可读宽度时才会被选中。不能通过压缩或截断入口标题来强行保留三列。

## 测试与验证

实施遵循测试驱动开发：先增加能够覆盖新摘要和入口行为的失败测试，再完成最小实现。

验证范围包括：

1. 标签和参与者数量变化后，摘要值正确更新。
2. 零条数据时显示零值摘要，三个入口仍然存在并可点击。
3. 三个入口继续导航到现有标签、参与者和图表页面。
4. 普通 Dynamic Type 且空间足够时，外部三列，三张卡片同排、同宽、同高，卡片内部为 vertical。
5. 普通 Dynamic Type 且空间不足时，外部完整单列，卡片内部为 horizontal；不出现两列状态或水平滚动。
6. Accessibility Dynamic Type 下，外部完整单列，卡片内部为 vertical；标题与摘要没有截断，也没有单字或两字宽的极窄文字列。
7. 中文和英文环境下入口标题与摘要均可读。
8. 浅色、深色和高对比度环境下卡片边界、文字和图标清晰。

代码完成后运行相关 Swift Package 测试和 Xcode 工程测试，并使用 XcodeBuildMCP 在模拟器中启动 App，分别检查正常宽度和辅助功能大字体状态，保存截图用于视觉核对。

## 不在本次范围内

- 修改标签、参与者或图表详情页的列表布局。
- 增加列表与网格切换设置或持久化展示偏好。
- 改变现有导航目标、深链格式或资料库导航栈。
- 在图表卡片中增加真实迷你图表、趋势计算或新的统计查询。
- 新增资料库入口、搜索、排序或编辑功能。

## 完成标准

- 资料库首页使用三张视觉一致、权重相同的卡片替代系统列表。
- 普通 Dynamic Type 且空间足够时，外部三列、卡片内部 vertical。
- 普通 Dynamic Type 且空间不足时，外部完整单列、卡片内部 horizontal。
- Accessibility Dynamic Type 下，外部完整单列、卡片内部 vertical，标题与摘要没有截断或极窄文字列。
- 页面不会出现两列布局、`2 + 1` 空位或宽度不一致的入口卡片。
- 标签与参与者数量实时更新，图表摘要不展示虚构数据。
- 三个入口的导航、深链和目标页面行为保持不变。
- Dynamic Type、VoiceOver、多语言、浅色和深色环境验证通过。
